# /raw-press

호출되면 아래 두 단계를 순서대로 수행한다.

## 1단계 — 옵시디언 작업 현황 업데이트
- 옵시디언 vault에서 **최종 저장(핸드오버/세이브) 시점**을 먼저 확인한다.
- 그 시점 **이후부터 지금까지의 작업 현황**만 추려 vault에 업데이트한다.
  - 우선 `obsidian-save` 스킬을 사용한다. 불가하면 `mcp-obsidian` 도구로 직접 append/patch 한다.
  - 보안 규칙 준수: 시크릿 평문 금지(환경변수 이름 + 위치만 기록).
- 업데이트한 파일 경로/섹션을 한 줄로 보고한다.

## 2단계 — 컨텍스트 압축
원하는 압축 방식:

> 마지막 작업은 전부 기억하고, 최근 작업부터 이전 작업 역순으로 거슬러 올라가면서 압축 비중을 줄이면서 진행.

해석: 가장 최근 작업은 **전체 보존(축약 없음)**, 시간 역순으로 거슬러 올라갈수록 **보존 분량(비중)을 점점 줄여** 오래된 작업일수록 핵심만 남긴다.

주의: Claude Code에서 커스텀 명령은 빌트인 `/compact`를 직접 실행할 수 없다. 실제 컨텍스트 압축은 사용자가 직접 `/compact`를 입력해야 발동된다. 따라서 1단계를 마친 뒤, 아래 한 줄을 사용자가 그대로 복사해 실행할 수 있도록 제시한다:

```
/compact 마지막 작업은 전부 기억하고, 최근 작업부터 이전 작업 역순으로 거슬러 올라가면서 압축 비중을 줄이면서 진행. (가장 최근=전체 보존, 과거로 갈수록 핵심만 남김)
```
