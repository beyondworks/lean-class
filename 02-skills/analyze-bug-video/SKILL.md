# Analyze Bug Video

모바일 UI 버그 녹화 영상을 분석하여 오류를 진단합니다.

## 사용법
```
/analyze-bug-video path/to/bug-recording.mp4
/analyze-bug-video path/to/video.mov 2 5    # 2~5초 구간만
```

## 워크플로우

### 1. 입력 파싱
- 첫 번째 인자: 영상 파일 경로 (필수)
- 두 번째 인자: 시작 시간(초) (선택, 기본: 0)
- 세 번째 인자: 종료 시간(초) (선택, 기본: 영상 끝)

### 2. 프레임 추출
영상 파일 옆에 `_frames/` 디렉토리를 만들고 프레임을 추출합니다.

```bash
# 영상 길이 확인
ffprobe -v error -show_entries format=duration -of csv=p=0 "<영상경로>"

# 프레임 추출 (1초 간격, 구간 지정 시 -ss/-to 적용)
ffmpeg -y -i "<영상경로>" [-ss <시작>] [-to <종료>] -vf "fps=1" "<출력경로>/frame_%03d.png"
```

- 영상이 10초 이하: fps=2 (초당 2프레임)
- 영상이 10~30초: fps=1 (초당 1프레임)
- 영상이 30초 초과: fps=0.5 (2초당 1프레임)
- 최대 프레임 수: 20장 (초과 시 fps 조정)

### 3. 프레임 분석
Read 도구로 추출된 프레임 이미지를 순서대로 읽습니다.

분석 관점:
- **레이아웃 깨짐**: 요소 겹침, 잘림, 밀림, 빈 공간
- **상태 전이 오류**: 로딩 → 에러, 깜빡임, 잘못된 화면 전환
- **스크롤/애니메이션**: 떨림, 점프, 부자연스러운 움직임
- **데이터 표시**: 빈 목록, 깨진 이미지, 잘못된 텍스트
- **터치 반응**: 버튼 미반응, 이중 실행, 딜레이

### 4. 진단 리포트 출력

아래 형식으로 결과를 보고합니다:

```
## 버그 진단

**발견 시점**: frame_003.png ~ frame_005.png (3~5초 구간)
**증상**: [구체적 시각 증상 설명]
**심각도**: 높음/중간/낮음
**추정 원인**: [기술적 원인 추정]

### 프레임 비교
- frame_002.png: 정상 상태 (버그 발생 직전)
- frame_003.png: [변화 설명]
- frame_005.png: [최종 상태]

### 수정 방향
- [구체적 수정 제안]
```

### 5. 코드 연결 (선택)
현재 프로젝트에 소스 코드가 있으면:
- 증상과 관련된 컴포넌트/파일을 Grep/Glob으로 탐색
- 의심되는 코드 위치 제시

## 주의사항
- 추출된 프레임은 분석 후 삭제하지 않음 (사용자가 직접 확인할 수 있도록)
- 영상 파일이 존재하는지 먼저 확인
- ffmpeg/ffprobe가 없으면 설치 안내
