# Pre-Deploy Review Team

배포 전 코드베이스 전체를 6개 관점에서 병렬 리뷰하는 에이전트 팀.
커맨드 하나로 코드 품질, 보안, UX, 빈 기능, 아키텍처, 접근성을 동시에 점검합니다.

## 워크플로우

```
[프로젝트 경로 입력]
         │
         ▼
┌─────────────────────────────────────────────┐
│  0. scout (정찰 에이전트)                      │
│     - 프로젝트 구조 파악 (프레임워크, 라우트)    │
│     - 기술 스택 식별                           │
│     - 리뷰 대상 파일 목록 생성                  │
│     - review-context.md 작성                  │
└─────────────────────────────────────────────┘
         │
         ▼
┌─────────────────────────────────────────────┐
│  1. 병렬 리뷰 (6개 에이전트 동시 실행)          │
│                                              │
│  [code-quality]     [security]    [ux-audit] │
│  로직 결함           OWASP Top10   UX 흐름    │
│  anti-patterns      시크릿 노출    사용성      │
│  SOLID 위반         인젝션 취약점   일관성      │
│  타입 안전성         인증/인가      반응형      │
│                                              │
│  [dead-feature]     [architecture] [a11y]    │
│  빈 기능 탐지        의존성 구조    WCAG 준수   │
│  미연결 핸들러       순환 참조      키보드 탐색  │
│  하드코딩 mock       확장성 병목    스크린리더   │
│  TODO/FIXME         에러 경계      색상 대비   │
└─────────────────────────────────────────────┘
         │
         ▼
┌─────────────────────────────────────────────┐
│  2. synthesizer (종합 에이전트)                │
│     - 6개 리포트 통합                         │
│     - 심각도별 정렬 (Critical → Low)          │
│     - 중복 제거 & 교차 관점 연결               │
│     - 최종 리포트 생성                        │
│     - 수정 우선순위 액션 플랜                  │
└─────────────────────────────────────────────┘
         │
         ▼
[review-report.md 최종 산출물]
```

## 사용법

```
/pre-deploy-review [프로젝트 경로]
/pre-deploy-review .
/pre-deploy-review /Users/me/my-app
```

## 산출물

| 파일 | 내용 |
|------|------|
| `review-context.md` | 프로젝트 구조 & 기술 스택 요약 |
| `review-code-quality.md` | 코드 품질 리포트 |
| `review-security.md` | 보안 취약점 리포트 |
| `review-ux.md` | UX 감사 리포트 |
| `review-dead-features.md` | 빈 기능 & 미완성 코드 리포트 |
| `review-architecture.md` | 아키텍처 평가 리포트 |
| `review-a11y.md` | 접근성 리포트 |
| **`review-report.md`** | **통합 최종 리포트 + 액션 플랜** |

## 심각도 등급

- **CRITICAL**: 배포 차단. 즉시 수정 필수 (보안 취약점, 데이터 손실 위험)
- **HIGH**: 배포 전 수정 강력 권장 (로직 결함, 빈 기능, 인증 누락)
- **MEDIUM**: 배포 가능하나 조기 수정 권장 (UX 문제, 코드 품질)
- **LOW**: 개선 사항 (스타일, 접근성 미세 조정, 리팩터링 제안)
