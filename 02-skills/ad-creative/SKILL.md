# 커머스 멀티 매체 퍼포먼스 광고 소재 생성 오케스트레이터 (Figma 네이티브, 4단계)

커머스 D2C 랜딩페이지를 입력받아 4개 매체(메타/구글/네이버/카카오) 광고 소재 한 세트를 **사용자 지정 Figma 파일에 직접 작성**하는 4단계 파이프라인입니다. 1~3단계는 서브에이전트가 담당하고, 마지막 4단계는 **오케스트레이터(메인 thread)가 직접 Figma MCP를 호출**합니다.

> **왜 4단계?** Subagent는 메인의 deferred MCP 도구 schema 로드 상태를 상속하지 않아 figma MCP 직접 호출이 실패합니다. 그래서 디자이너는 Plugin API JS 코드만 산출하고, 메인 thread가 그 코드를 `mcp__figma-remote-mcp__use_figma`로 실행하는 분업으로 안정화했습니다.

## 지원 매체 4종

| 매체 ID | 매체명 | 인식 키워드 |
|---------|--------|-------------|
| `meta` | 메타 (Facebook / Instagram) | 메타 / FB / IG / 인스타 / 페이스북 |
| `google` | 구글 디스플레이 + 디맨드젠 | 구글 / Google / GDN / RDA / 디맨드젠 / Demand Gen |
| `naver` | 네이버 성과형 디스플레이 (GFA) | 네이버 / NAVER / GFA / 성과형 DA |
| `kakao` | 카카오 모먼트 디스플레이 | 카카오 / Kakao / 모먼트 / DDN |

매체별 정확한 spec(사이즈·글자수·안전영역·CTA)은 [.claude/](.claude/) — 카피라이터·디자이너가 매번 자동 Read.

## 파이프라인

```
[입력: 랜딩 URL/본문 + Figma URL + 매체(선택)]
         │
         ▼
1. ad-landing-analyzer  (Agent 도구)
   → 9축 분석 결과 + 이미지 인벤토리 (마크다운)
   → 매체 무관 동일 로직, 매체별 카피 가이드는 메모로 추가
         │
         ▼
2. ad-copywriter  (Agent 도구)
   → 카테고리 매칭 + 디자인 brief A/B + 본문 카피 3종 (자연어)
   → 매체별 글자수·CTA 자동 적용
         │
         ▼
3. ad-designer  (Agent 도구)
   → output/<timestamp>/figma-plugin.js
   → 매체별 frame 사이즈 모두 또는 필수만 자동 생성
   → 디자인 결정 노트 + 실행 안내
         │
         ▼
4. Figma 적용 — 메인 thread 직접 실행
   → Read로 figma-plugin.js 본문 추출
   → mcp__figma-remote-mcp__use_figma(fileKey, code) 호출
   → 반환된 pageId/frame ids로 Figma 링크 생성
         │
         ▼
[산출: Figma 링크 + 본문 카피 팩 + 결정 노트]
```

## 입력 검증

다음을 **사용자에게 한 번만** 확인:

1. **랜딩페이지** — URL 또는 본문 텍스트 (필수). **커머스 D2C 랜딩 한정** — 9개 카테고리 (패션·뷰티·건기식·식품·가전·디지털·리빙·다이어트·펫·키즈). 비커머스(서비스·B2B·로컬·교육 강의·의료·여행 등)면 분석기가 "본 하네스 범위 외"를 안내하고 사용자 확인 요청.
2. **Figma 파일 URL** — `figma.com/design/<fileKey>/...` 형식. **매번 필수**. 디폴트 저장 안 함.
3. **광고 매체** — 다음 중 하나 (선택):
   - `meta` (메타·FB·IG·인스타) — **디폴트 매체**
   - `google` (구글 디스플레이·RDA·디맨드젠)
   - `naver` (네이버 성과형 DA·GFA)
   - `kakao` (카카오 모먼트 DA)
   - **미지정 시 → 메타 IG 피드 1:1 (1080×1080)** 단독 기준 (디폴트 변경: 4:5 → 1:1)
4. **사이즈 옵션** (선택):
   - **"필수 사이즈"** / **"대표 사이즈"** → 매체별 필수 ★ 사이즈만
   - **"모든 사이즈"** → 매체에서 지원하는 사이즈 전체
   - 명시 없으면 → **필수 사이즈** 기본
5. **광고 목적** — 전환/리드/도달 (선택, 미지정 시 "전환")
6. **추가 컨텍스트** — 강조 포인트, 금지어, 브랜드 톤 (선택)

이미 사용자가 메시지에서 위 정보를 충분히 줬으면 추가 질문 없이 진행. 하나라도 부족하면 한 번에 묶어서 묻기.

**fileKey 추출 규칙:**
- `https://www.figma.com/design/<KEY>/<name>?node-id=...` → fileKey = `<KEY>`
- `https://figma.com/design/<key>/branch/<branchKey>/<name>` → fileKey = `<branchKey>`
- 다른 형식이면 사용자에게 다시 확인.

**매체 인식 패턴:**
- "메타" / "FB" / "IG" / "인스타" / "페이스북" 단어 → `meta`
- "구글" / "Google" / "GDN" / "RDA" / "디맨드젠" / "Demand Gen" → `google`
- "네이버" / "NAVER" / "GFA" / "성과형" → `naver`
- "카카오" / "Kakao" / "모먼트" / "DDN" → `kakao`
- 매체 명시 없으면 → `meta`
- 매체 명시 + 사이즈 명시 (예: "구글 RDA 1.91:1") → 그 사이즈 단독 제작
- 매체 명시 + "모든 사이즈" / "필수 사이즈" → 매체별 사이즈 표 따름

## 매체별 디폴트 사이즈

매체 명시되었으나 사이즈 미명시 시:

| 매체 | 디폴트 사이즈 | 추가 옵션 (사용자가 "모든 사이즈" 시) |
|------|--------------|-------------------------------------|
| `meta` (디폴트) | **IG 피드 1:1 (1080×1080)** | 4:5 (1080×1350) / 9:16 스토리 (1080×1920) / 1.91:1 (1080×566) |
| `google` (RDA + Demand Gen) | **1.91:1 (1200×628) + 1:1 (1200×1200) 동시** | + 4:5 (960×1200, Demand Gen만) / 로고 1:1·4:1 |
| `naver` (GFA) | **이미지 배너 1250×560 + 정방형 1:1 (1200×1200) + 피드 16:9 (1200×628) 동시** | + 광각형 (1200×680) |
| `kakao` (모먼트 DA) | **네이티브 1:1 (1080×1080) + 2:1 (1200×600) 동시** | + 9:16 (720×1280) / 4:5 (960×1200) |

→ 매체별 *필수 사이즈*는 동시에 모두 제작 (한 figma 페이지에 여러 frame).

## 실행 순서

### Step 1. 랜딩 분석
`ad-landing-analyzer` 에이전트를 Agent 도구로 호출. URL/본문/추가 컨텍스트를 prompt에 그대로 전달. 분석 결과(9개 축, 마지막 축은 **이미지 자산 인벤토리**)를 마크다운으로 받아서 그대로 보존.

분석은 **매체 무관 동일 로직**. 다만 prompt에 *광고 매체*를 메모로 포함해서 분석기가 매체 특성에 맞는 가이드(카피 글자수·시각 강도)를 카피라이터에게 메모로 흘릴 수 있게 함.

### Step 2. 카피 작성
`ad-copywriter` 에이전트를 Agent 도구로 호출. prompt에:
- 1단계 분석 결과 전체 (이미지 인벤토리 포함)
- 사용자 지시 톤·금지어·강조점
- **광고 매체** (사용자 입력 또는 디폴트 `meta`)
- **사이즈 옵션** (필수 / 모든)
- 다음 가이드 추가:

- **"매번 다른 변주로 작성하세요. 이전 호출과 다른 훅 형태(숫자/의문/명령/대비 등)를 시도하세요."**
- **"랜딩 이미지는 참고용 정보로만 흘리세요. 광고 컨셉과 합치할 때만 [이미지 디렉션]에 한 줄 메모하고, 별도 '디자이너에게 — 랜딩 이미지 참고' 섹션으로 분석 표를 패스스루하세요. 강제 사용 X."**
- **"A/B 축은 4가지(이미지/카피/소구/CTA) 중 의식적으로 하나 선택. *소구 축이면 [이미지 디렉션]도 두 안 사이에 충실하게 다르게* 작성. 의사 A/B 금지. 각 안의 [이미지 디렉션]에 *dominant 컬러 톤* 반드시 명시."**
- **"매체가 `naver` 또는 `kakao`면 헤드라인 15자 이하 강한 hook 필수 (가장 빡빡). 매체가 `meta` IG 1:1이면 12자 이하 강한 hook 권장. 매체가 `google` Demand Gen·RDA면 30~40자 가능하지만 첫 5단어가 hook을 결정."**

카피 결과(자연어 brief A/B + 본문 3종 + 랜딩 이미지 참고 표)를 그대로 보존.

### Step 3. 디자인 코드 산출
`ad-designer` 에이전트를 Agent 도구로 호출. prompt에:
- 2단계 카피 결과 전체 (자연어 brief A/B + 본문 3종 + **디자이너에게 — 랜딩 이미지 참고** 표 패스스루)
- **Figma fileKey** (사용자 입력에서 추출)
- **광고 매체** + **사이즈 옵션** (필수 / 모든)
- 랜딩 URL·카테고리·timestamp (없으면 디자이너가 Bash로 생성)
- 다음 가이드 추가:

- **"매체가 `meta` 9:16 스토리면 상단 14% / 하단 20% 안전영역 비우고 텍스트 영역 중간 66%에 집중. placeholder는 frame 0~H 풀블리드."**
- **"매체가 `google`(RDA·Demand Gen)이면 동일 카피·이미지로 *여러 frame 동시 생성* (1.91:1 + 1:1 동시). 같은 페이지 안에 frame 여러 개 좌→우 배치."**
- **"매체가 `naver`면 이미지 배너 1250×560 + 네이티브 16:9 (1200×628) + 정방형 1:1 (1200×1200) 동시 생성."**
- **"매체가 `kakao`면 네이티브 1:1 (1080×1080) + 2:1 (1200×600) 동시 생성."**
- **"각 frame마다 같은 카피·이미지지만 사이즈에 맞춰 텍스트 위치·크기 자동 조정. 1.91:1 가로형은 헤드라인 좌측·이미지 우측 / 1:1·4:5는 상하단 anchor / 9:16은 중간 66% 안 anchor."**
- **"각 안([이미지 디렉션]·[톤·무드])의 dominant tone을 4가지(다크/라이트/컬러풀/중간) 중 하나로 판정 → 톤별 매트릭스로 텍스트·scrim·배지·CTA 컬러를 각 안마다 별도로 결정."**
- **"랜딩 이미지 참고 패널을 별도 영역으로 figma-plugin.js에 작성. placeholder에 강제 합성 X."**
- **"8개 일반 원칙 모두 자동 적용 (setRangeFills 인덱스 맵 / 보수적 컬러 통일 / 좌표 충돌 동적 GAP / 광고 vs 스크린샷 단순화 / 인과 트리오 / CTA 텍스트 width = 박스 전체 / 실사 합성 가독성 보수적 시작 / 사진 z-order 무관 가독성 가드)."**

디자이너는 `output/<timestamp>/figma-plugin.js`를 작성하고 응답에 파일 경로·디자인 노트·실행 안내를 반환. **이 단계에서는 Figma에 아무것도 만들어지지 않음.**

### Step 4. Figma 적용 (메인 thread)

오케스트레이터가 직접 다음을 수행:

1. `Read`로 `output/<timestamp>/figma-plugin.js` 파일을 읽기.
2. 코드 본문 추출:
   - 코드가 `(async () => { ... })()` IIFE면 그대로 use_figma의 code에 전달
   - **`figma.closePlugin()`, `figma.notify()`가 있으면 제거**
   - **`figma.currentPage = page` 직접 할당이 보이면 `await figma.setCurrentPageAsync(page);`로 교체**
   - 마지막에 `return { pageId, pageName, frames }` 형태로 ID 반환하도록 보완
3. `mcp__figma-remote-mcp__use_figma` 호출:
   - `fileKey`: 사용자가 준 fileKey
   - `description`: "Create AD page for <slug> [<매체>]"
   - `code`: 위에서 정리한 코드
4. 반환된 `{ pageId, frames }`로 Figma 링크 생성:
   - 페이지: `https://www.figma.com/design/<fileKey>/?node-id=<pageId의 ":"를 "-"로>`
   - 각 프레임: 동일 형식
5. 호출 실패 시(권한·인증·MCP 한도 등):
   - 에러 메시지 그대로 사용자에게 보고
   - figma-plugin.js 경로 + manifest.json 경로 안내
   - "Figma 데스크톱에서 Plugin import 후 수동 실행" fallback 가이드

```
- 매체 변경: "구글 RDA로 다시" / "네이버 GFA로 추가" / "카카오 모먼트로 변환"
- 다른 커머스 카테고리: "[카테고리 이름 — 패션/뷰티/건기식/식품/가전·디지털/리빙/다이어트/펫/유아·키즈]으로 다시"
- 이미지 합성: placeholder 자리에 사진 첨부 → upload_assets로 fill 채움
```

## 부분 재실행

사용자가 "카피만 다시", "디자인만 다시", "B안만 다크톤으로", "구글 매체로 변환" 등을 요청하면:

- **분석 결과는 재사용** (이미 가지고 있음)
- 해당 단계 에이전트만 다시 호출
- 카피만 재생성: copywriter prompt에 "이전 결과와 *완전히 다른* 훅 형태로 다시 시도" 명시
- 디자인 부분 수정: designer prompt에 "수정 모드 — 기존 Figma 페이지의 해당 노드만 업데이트(새 페이지 생성 금지). 가장 최근 `AD - ...` 페이지를 찾아 노드 검색 후 변경." 명시. Step 4에서 메인이 그 코드를 다시 use_figma로 실행.
- 디자인 전체 다시: designer prompt에 "신규 페이지로 다시 생성" 명시.
- **매체 변경 (예: 메타 → 구글 RDA)**: 분석 결과 재사용 + 카피라이터부터 다시 호출 (매체 글자수 변경되므로). 디자이너는 새 매체 frame 사이즈로 재생성 (새 페이지).

부분 수정 시에도 fileKey는 사용자에게 다시 묻지 않고 직전 호출의 fileKey를 그대로 재사용.

## 주의

- 절대 본인이 직접 카피·디자인을 작성하지 마세요. 1~3단계는 모두 서브에이전트 위임.
- **Step 4(Figma 적용)는 메인 thread만 수행**. Subagent에 절대 위임하지 마세요. Subagent는 deferred MCP 도구를 못 부릅니다.
- 단계별 결과를 사용자에게 *축약*해서 전달하지 마세요. 각 에이전트의 출력 전체를 그대로 보여줘야 사용자가 어디를 수정할지 결정할 수 있습니다.
- 에이전트 호출이나 Figma 적용이 실패하면 실패 사실을 그대로 보고하고, fileKey 재확인 또는 수동 실행 fallback 안내.
- 사용자가 추가 정보 없이 "광고 만들어줘"만 보내고 랜딩/Figma URL이 없으면, 위 입력 검증 항목을 한 번에 물어보세요.
- **HTML/PNG는 산출하지 않습니다.** 디자인은 항상 Figma에 직접. 사용자가 "PNG로 뽑아줘" 명시 시 Figma에서 export 하도록 안내.
- **매체별 글자수 차이가 큼** — 네이버·카카오 15자 / 메타 27~40자 / 구글 30~40자. 카피라이터가 매체 입력에 맞춰 자동 조정 의무.
- **다중 매체 동시 작업** — 사용자가 "메타 + 구글 둘 다"라고 명시하면 두 매체 모두 생성 (가장 빡빡한 글자수 기준). 단일 호출 한 페이지 안에 매체별 영역 분리.
