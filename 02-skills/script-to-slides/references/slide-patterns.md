# 슬라이드 패턴 카탈로그

대본 섹션을 어떤 패턴에 배정할지 결정표. 스니펫은 `index-template.html`에 패턴별 1개씩 들어있다.

| 패턴 | 언제 | 클래스 | 비고 |
|---|---|---|---|
| **표지** | 시작/장 전환 | `.overlay.center` + `.title` | 이미지 배경 + `.scrim`. 로고 원하면 `.brand` |
| **개념 예고** | "오늘 N가지" 예고 | `.pills > .pill` | 용어 → 한 줄 뜻 |
| **다이어그램** | 구조/관계/흐름 | `.overlay.split` + `<svg class="dg">` | 좌 텍스트 / 우 SVG. `svg-diagrams.md` |
| **개념 정의** | 한 개념 깊게 | `.gcard` (글라스 패널) | 제목 `.ct` + 본문 `.cb` |
| **Before/After** | 대비/리프레임 | `.cmp > .pan.bad/.good` | 끝에 `.takeaway` 한 줄 |
| **3·4-up 카드** | 병렬 항목 3~4개 | `.cards3` / `.cards4` | 메커니즘·기능·옵션 |
| **정리(맵)** | 항목 → 요약 나열 | `.maplist > li(.f/.a/.t)` | 회고·매핑 |
| **상세페이지** | 스펙·전환·가격·커리큘럼 | `.slide.detail` | 사진X 플랫다크, 고밀도, 상단정렬 |
| **흐름 6단계** | 파이프라인/단계 | `.flow6 > span + i` | 가로 칩 흐름 |
| **FAQ** | 질문 선제 차단 | `.faq > li(b+span)` | 전환부 필수 |
| **CTA** | 행동 유도 | `.cta` + `.cta-sub` | |

## 배정 원칙
- **한 슬라이드 = 한 메시지.** 2가지 이상이면 쪼갠다.
- 추상 개념은 **쉬운 비유 슬라이드**를 한 장 덧붙인다(예: "컨텍스트=책상", "스킬=로봇 기능"). 입문자 체감↑.
- 본문(개념) = 시네마틱(이미지+스크림), 전환/스펙 섹션 = `.detail`(플랫다크 고밀도). 시각적으로 섹션을 가른다.
- 정보가 많으면 디자인 일관성보다 **전달**이 우선 — 카드·폰트 축소(`.detail`가 이미 축소 토큰 보유), 단 도트·버튼과 겹치지 않게 `.detail`의 `padding-bottom` 유지.

## 컴포넌트 빠른 참조 (theme.css)
- 강조: `<span class="g">…</span>` (그린). 제목 2줄 = 흰 + 그린.
- 떠 있는 칩(공감/예시): `.chips > .chip`.
- 다이어그램 텍스트: `.dg .l`(라벨 19px) · `.dg .s`(보조 13.5px) · `.nd`(노드) · `.na`(강조노드) · `.e`/`.ea`(엣지/강조엣지).
- 토큰: `:root` 의 `--bg`·`--green`·`--font`만 프로젝트별로 바꾼다.
