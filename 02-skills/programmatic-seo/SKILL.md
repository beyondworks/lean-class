# Programmatic SEO

Build SEO-optimized pages at scale using templates and data.

## Initial Assessment

Before designing a pSEO strategy:

1. **Business Context**: Product/service? Target audience? Conversion goal?
2. **Opportunity**: What search patterns exist? How many pages? Volume distribution?
3. **Competition**: Who ranks now? What do their pages look like?

---

## Core Principles

### 1. Unique Value Per Page

Every page must provide specific value—not just swapped variables.

### 2. Proprietary Data Wins

Hierarchy of defensibility:

1. Proprietary (you created it)
2. Product-derived (from users)
3. User-generated (community)
4. Licensed (exclusive access)
5. Public (weakest)

### 3. Clean URL Structure

**Always subfolders, not subdomains:**

- ✅ `yoursite.com/templates/resume/`
- ❌ `templates.yoursite.com/resume/`

### 4. Genuine Search Intent Match

### 5. Quality Over Quantity

100 great pages > 10,000 thin pages

### 6. Avoid Google Penalties

No doorway pages, keyword stuffing, or duplicate content.

## Implementation Framework

### 1. Keyword Pattern Research

- Identify repeating structure and variables
- Validate aggregate volume and distribution
- Check trend direction

### 2. Data Requirements

- What populates each page?
- First-party, scraped, licensed, or public?
- How is it updated?

### 3. Template Design

**Page structure:**

- Header with target keyword
- Unique intro (not just variable swap)
- Data-driven sections
- Related pages / internal links
- Intent-appropriate CTAs

**Ensuring uniqueness:**

- Conditional content based on data
- Original insights per page

### 4. Internal Linking Architecture

**Hub and spoke:**

- Hub: Main category page
- Spokes: Individual pSEO pages
- Cross-links between related spokes

Breadcrumbs with structured data.

### 5. Indexation Strategy

- Prioritize high-volume patterns
- Noindex very thin variations
- Manage crawl budget
- Separate sitemaps by page type

## Quality Checklist

### Pre-Launch

- [ ] Each page provides unique value
- [ ] Answers search intent
- [ ] Unique titles and meta descriptions
- [ ] Schema markup implemented
- [ ] Connected to site architecture
- [ ] In XML sitemap

### Post-Launch Monitoring

Track: Indexation rate, Rankings, Traffic, Engagement, Conversion

Watch for: Thin content warnings, Ranking drops, Manual actions

---

## Common Mistakes

- **Thin content**: Just swapping city names
- **Keyword cannibalization**: Multiple pages same keyword
- **Over-generation**: Pages with no search demand
- **Poor data quality**: Outdated information
- **Ignoring UX**: Pages exist for Google, not users
